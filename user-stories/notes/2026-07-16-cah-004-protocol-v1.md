# CAH-004 protocol version 1 implementation note

## Outcome

CAH-004 turns the CAH-003 child pipes into a strict protocol version 1 boundary. The Ink parent now
sends a validated initialization command, Python confirms the same canonical workspace, and the TUI
enters `running` only after a correlated `runtime.ready`. Normal cleanup sends
`runtime.shutdown` before stdin close and retains the process-group signal fallback.

This unit defines and validates the session command and event families but does not start, cancel,
stream, or reduce a session. CAH-005 and CAH-006 own those behaviors.

## Locked contract

- Every message is one UTF-8 JSON object followed by exactly one LF.
- CRLF, blank lines, embedded physical newlines, invalid UTF-8, oversized lines, and incomplete EOF
  are framing failures.
- The byte bound is 64 KiB per physical line, excluding LF.
- Timestamps use exact `YYYY-MM-DDTHH:mm:ss.SSSZ` UTC form with years 0001 through 9999.
- Command and correlation IDs match `cmd_[A-Za-z0-9_-]{1,64}`.
- Session IDs match `ses_[A-Za-z0-9_-]{1,64}`.
- Session sequence values are positive and no larger than `9007199254740991`.
- Pydantic and Zod objects are strict; unknown envelope or payload fields are invalid.
- Error codes are bounded machine values, visible failure messages exclude terminal controls, and
  both encoders reject an object whose UTF-8 representation exceeds 64 KiB.
- A change to required fields, meanings, or ordering requires a new protocol version. An additive
  field still requires coordinated readers, writers, and fixtures before use.

Payloads remain deliberately small:

| Message | Payload |
| --- | --- |
| `runtime.initialize` / `runtime.ready` | `workspace` |
| `session.start` | `task` |
| `session.cancel` | `session_id` |
| `runtime.shutdown` | Empty |
| `session.started` / `session.completed` / `session.cancelled` | Empty |
| `assistant.delta` / `assistant.completed` | `text` |
| `session.failed` | `code`, `message` |
| `runtime.error` | `code`, `message`, `recoverable` |

## Implementation map

- `src/code_assist_harness/protocol/models.py` defines frozen Pydantic v2 wire models.
- `src/code_assist_harness/protocol/codec.py` performs JSON, version, envelope, discriminator, and
  selected-model validation without returning raw input or validator details.
- `src/code_assist_harness/protocol/streams.py` implements bounded readers and the ordered event
  writer.
- `tui/src/protocol.ts` defines strict Zod schemas, safe parser results, and validated encoders.
- `tui/src/protocol-stream.ts` implements byte framing and fatal UTF-8 decoding.
- `protocol/fixtures/v1/manifest.json` lists 12 valid and 12 invalid reviewed fixtures consumed by
  both languages.
- `src/code_assist_harness/runtime.py` contains malformed command lines, initializes one workspace,
  reports deferred session commands safely, and honors shutdown.
- `tui/src/runtime-supervisor.ts` owns initialization correlation, readiness timeout, fail-closed
  event handling, shutdown framing, and child reaping.

## Failure policy

The failure direction matters:

- Bad stdin is a failed command. Python emits a safe, recoverable `runtime.error`, discards only that
  physical line, and continues. This lets a later valid initialization succeed.
- Bad stdout threatens the authoritative UI event stream. The TUI preserves the distinction between
  `unknown_type`, `invalid_payload`, framing, and readiness mismatch, enters `protocol-failed`, and
  closes command input. It never guesses or renders the untrusted object.
- A valid `runtime.error` correlated to initialization becomes a controlled startup failure.
- A valid event that is not legal in the current lifecycle state becomes `unexpected_event`; valid
  post-readiness errors are sanitized before the TUI displays the fail-closed state.
- A `runtime.ready` with the wrong command ID or workspace becomes `readiness_mismatch`.
- Spawn without readiness becomes `readiness_timeout`; spawn itself is no longer evidence that
  Python accepts protocol commands. The deadline starts before the initialization write callback,
  so backpressure cannot suspend startup forever.

No safe error includes the original line, Pydantic input, Zod issue detail, environment value, or
unbounded output.

## Implementation discoveries

### JSON number semantics differ across runtimes

JavaScript parses `1`, `1.0`, and `1e0` as the same number. Python's JSON decoder normally preserves
integer and float lexemes as different runtime types, which conflicted with strict Pydantic models.
The Python byte codec now applies finite JavaScript-number semantics and converts an integral parsed
number to `int`; direct Python `validate_*` calls remain strict and reject caller-supplied floats.
This keeps byte-level parity without a lexical JSON scanner. Both codecs also classify a numeric
literal that overflows to a non-finite value as `malformed_json`.

### Sequence commit must survive cancellation

The writer lock covers sequence allocation through sink completion. If a sink completes while the
producer task is being cancelled, the writer commits that sequence before propagating cancellation;
otherwise the next event could reuse a sequence already present on stdout. A sink failure before
completion does not advance the sequence.

### Workspace identity has two owners for different reasons

The CLI argument fixes the process security boundary before stdin is trusted. The
`runtime.initialize` payload then proves that the protocol peer agrees with that exact canonical
workspace. Python resolves the payload again and emits a nonrecoverable `workspace_mismatch` rather
than allowing protocol data to redirect the process.

A schema-valid string is not necessarily a filesystem-valid path. NUL-containing values can make
`Path.resolve()` raise `ValueError`, so workspace resolution contains `ValueError` alongside OS and
resolution failures. A regression proves NUL and lone-surrogate inputs become safe protocol errors
without raw bytes or tracebacks reaching stdout.

### Framing precedes JSON classification

CRLF is rejected by the byte reader even though many JSON parsers accept a trailing carriage return
as whitespace. Two concatenated objects on one LF line remain `malformed_json`; the implementation
does not add a fragile structural scanner merely to reinterpret the JSON parser's failure.

## Validation evidence

- `uv run --no-cache --no-sync --frozen --offline pytest` passes 120 tests.
- `uv run --no-cache --no-sync --frozen --offline ruff check .` passes.
- `uv run --no-cache --no-sync --frozen --offline ruff format --check .` passes.
- `npm --prefix tui run check` passes 119 tests plus TypeScript and ESLint checks.
- Python and TypeScript accept all 12 valid fixtures and reject all 12 invalid fixtures at the named
  boundary stage.
- The Python subprocess test sends malformed, unsupported, unknown, and invalid-payload lines before
  valid initialization; every stdout line remains a validated v1 event and readiness still occurs.
- The real Node-to-uv-to-Python test observes the readiness handshake, confirms poisoned Python and
  uv selectors do not redirect the child, sends orderly shutdown, and proves both processes are
  reaped through `/proc`.

All default evidence is model-free, credential-free, and network-free.

## Follow-up

CAH-005 can now connect task input to `session.start`, emit deterministic session events through the
existing ordered writer, and reduce validated events into visible streaming state. It must not
weaken the protocol failure policy or allow React components to decide session authority.
